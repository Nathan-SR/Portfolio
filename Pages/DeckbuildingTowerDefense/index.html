Deckbuilding Tower Defense in Godot
A hybrid of the tower-defense and deck-building roguelike genres. The core of the game is flexibility: players can design almost any tower they can imagine—provided they play their cards right.

I began this project simply to get familiar with a game engine. Over time, it’s grown into much more: a playground for experimentation, a testbed for new systems, and a collaborative codebase built with two close friends. It’s been my first experience designing and maintaining a large game project from the ground up with others.

The project is still in active development, but below are a few features I’m most proud of.
Component-Based Towers
Mixed entity-component system and strategy pattern allows for extremely modular towers. Players can mix upgrades to create unique towers.
Card System
Data-driven JSON-like card loading for easy extension and analysis.
Command pattern for card actions allows for easy addition of drop-in effects, trivializing the effort required to add many new cards.
Other Notables:
1. Fine-Tuned A*
Custom weights that minimize path divergence; path responsive to player for dynamic maze-building.

2. Agile Development with Youtrack
Created YouTrack dashboard to track progress on features and assign work. Managed multiple branches on GitHub to ensure code safety.

3. Main Map Generation
Created map generation algorithm to simulate a Slay the Spire-like progression map.
